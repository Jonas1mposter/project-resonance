## 1. 核心贡献

| 贡献 | 内容 |
|------|------|
| **数据集** | MDSC（普通话构音障碍语音库），18,630 条录音，17 小时，21 位患者 + 25 位健康对照 |
| **系统** | 三阶段渐进式唤醒词检测系统（SIC → SID → SDD） |
| **发现** | 说话人相关模型仅需 3 分钟注册语料即可大幅提升性能 |

## 2. 实验方法论详解（可迁移部分）

### 2.1 数据采集协议

| 参数 | MDSC 规范 | **迁移到 Project Resonance 的建议** |
|------|-----------|--------------------------------------|
| 采样率 | 16 kHz | ✅ 与我们标准一致 |
| 录音距离 | ~20 cm | ✅ 与我们标准一致（15~30cm） |
| 环境 | 安静室内 | ✅ 一致 |
| 唤醒词 | 10 个，每个重复 5 遍 | **可借鉴**：每条短语录 5 遍的策略 |
| 非唤醒词 | 355 条（含固定指令、自由指令、家居指令） | **可借鉴**：OOV 语料多样性设计 |
| 文本列表 | 295 条不重复语句/人 | 参考我们的 9 大类词表设计 |

### 2.2 可懂度评估框架（⭐ 高度可迁移）

论文使用 **双维度评估**，这是最值得借鉴的方法：

#### 主观可懂度（Subjective Intelligibility）
```
5 位专家标注员 → 各自转录录音 → 与标准文本对比
→ 计算每人准确率 → 取 5 人平均值 = 可懂度分数
```

**迁移建议**：
- 我们当前使用 1~5 分主观评分制，可增加一个**转录准确率**维度
- 将两种评分结合，提升标注一致性

#### 客观可懂度（Objective Intelligibility）
```
开源 ASR 模型（Paraformer / WeNet）→ 转录患者语音
→ 计算 WER（词错误率）= 客观可懂度
```

**迁移建议**：
- 用 StepFun ASR 对每条录音做自动转录
- 计算 CER（字错误率，中文用字代替词）
- 将 CER 作为自动化的可懂度指标，与人工评分交叉验证

### 2.3 三阶段渐进式训练（⭐⭐ 核心方法论）

```
阶段 1: SIC（Speaker-Independent Control）
  ├─ 数据：健康对照组训练集（C-train）
  ├─ 数据增强：频谱掩蔽 + 速度扰动 + 白噪声
  └─ 输出：通用基线模型

阶段 2: SID（Speaker-Independent Dysarthria）
  ├─ 基础：SIC 模型权重
  ├─ 数据：构音障碍训练集（D-train）+ 数据增强
  └─ 输出：通用构音障碍模型

阶段 3: SDD（Speaker-Dependent Dysarthria）
  ├─ 基础：SID 模型权重
  ├─ 数据：个人注册语料（3 分钟）+ 数据增强
  └─ 输出：个人定制化模型
```

**性能提升链**（D-test 上的 Score↓）：

| 模型 | FAR↓ | FRR↓ | Score↓ | 相对提升 |
|------|------|------|--------|---------|
| SIC | 0.1630 | 0.3708 | 0.5339 | — |
| SID | 0.1538 | 0.1875 | 0.3413 | **36%** |
| SDD | 0.0555 | 0.0833 | 0.1388 | **74%** |

### 2.4 数据增强策略（⭐ 直接可用）

| 策略 | 参数 | 适用场景 | **迁移优先级** |
|------|------|----------|---------------|
| **频谱掩蔽** | 频率 + 时间随机遮盖 | 增强鲁棒性 | 🟢 高 — 训练时可直接应用 |
| **速度扰动** | 比率 [0.9, 1.1] | 模拟说话速度变化 | 🟢 高 — 构音障碍患者语速差异大 |
| **白噪声** | SNR [-15, 15] dB | 模拟环境噪声 | 🟡 中 — 我们要求 SNR ≥ 15dB 的干净录音 |

### 2.5 数据集划分方法

| 维度 | MDSC 做法 | **迁移建议** |
|------|-----------|-------------|
| 说话人级别 | 说话人间无重叠 | ✅ 我们已采用说话人内划分 |
| 注册语料 | 每人预留 3 分钟（不入任何子集） | **新增**：为每位说话人预留注册集 |
| 正负样本比 | 唤醒词:非唤醒词 = 1:5 | **借鉴**：OOV 与词表内样本比例 |
| 性别平衡 | 测试集中男女平衡 | ✅ 我们已要求 3:7~7:3 |

### 2.6 注册语料实验（⭐ 极高参考价值）

论文对 **注册语料时长** 和 **正负样本比** 做了消融实验：

#### 正负样本比实验
```
1:0 → 1:1 → 1:2 → 1:3 → 1:4 → 1:5 → ... → 1:8
                                    ↑
                              最佳平衡点 (1:5)
```
- 纯正样本（1:0）效果最差
- 加入负样本后性能稳步提升
- **1:5 为最佳比例**，之后改善趋于平缓

#### 注册语料时长实验
```
1 min → 1.5 min → 2 min → 2.5 min → 3 min
  ↓                                    ↓
Score≈0.39                        Score≈0.14
```
- **3 分钟即可达到良好效果**
- 1 分钟到 3 分钟性能提升约 64%

**迁移建议**：
- 为每位说话人设计**最少 3 分钟的个性化注册环节**
- 注册语料中包含唤醒词/目标短语 + 5 倍量的非目标语料
- 这直接指导我们的数据采集 SOP §8

## 3. 评估指标体系的迁移

### 3.1 论文使用的指标

| 指标 | 定义 | 对应我们的指标 |
|------|------|---------------|
| FAR（误唤醒率） | 非唤醒词被错误识别为唤醒词 | ≈ False Acceptance Rate |
| FRR（拒绝率） | 唤醒词未被识别 | ≈ False Rejection Rate |
| Score = FAR + FRR | 综合评分（越低越好） | 可新增为综合指标 |

### 3.2 迁移到 Project Resonance

| 论文指标 | 我们的对应 | 建议调整 |
|----------|-----------|---------|
| Score = FAR + FRR | 无直接对应 | **新增**：综合误差分 = FAR + FRR |
| 可懂度-性能相关性分析 | 无 | **新增**：按 severity 分组报告性能 |
| 个体级性能报告（D1~D6） | 无 | **新增**：每位说话人独立评测报告 |

## 4. 关键发现与启示

### 4.1 可懂度与性能强正相关
- 可懂度越高，识别性能越好（符合直觉）
- **但中度患者(D3/D4)从微调中获益最大**（51%/67% 提升）
- 极重度患者(D6)即使个人定制仍效果有限

**启示**：我们的 MVP 应优先服务**轻度~中度**患者，这是投入产出比最高的群体。

### 4.2 个性化是关键方向
- 说话人间差异极大（音高、语速、呼吸模式、句边界）
- 通用模型对高可懂度者可能反而变差（D1 的 SID 退化）
- **3 分钟个人数据 + 微调 → 74% 性能提升**

**启示**：验证了我们 LoRA 微调路线的合理性，且数据需求量很低。

### 4.3 数据增强不可或缺
- 频谱掩蔽 + 速度扰动 + 噪声注入三件套
- 在数据稀缺场景下效果显著

**启示**：即使在模板匹配阶段，也应在特征提取前应用数据增强。
