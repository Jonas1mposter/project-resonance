# 产品需求文档 (PRD)

## 项目信息

- **项目名称**：共鸣 (Resonance)
- **版本**：v2.0
- **最后更新**：2026-04-13

## 背景与目标

共鸣是面向构音障碍（Dysarthria）患者的语音辅助沟通系统。目标是通过 AI 语音识别技术，帮助言语障碍患者更高效地表达日常需求。

### 目标用户

- 构音障碍患者（轻度至重度）
- 患者的照护人员
- 康复治疗师

### 核心问题

构音障碍患者的语音清晰度降低，普通 ASR 系统识别准确率极低。需要针对性优化的语音识别方案。

## 功能需求

### P0 — 核心功能

1. **语音识别**
   - 支持自部署 Whisper ASR 引擎
   - 离线自动降级到浏览器原生识别
   - 返回 Top-3 候选结果供用户选择

2. **短语管理**
   - 10 大生活场景分类预设短语
   - 自定义短语增删改
   - 短语导入导出

3. **语音合成反馈**
   - CosyVoice TTS 支持声音克隆
   - Web Speech API 降级方案

### P1 — 重要功能

4. **录音训练**
   - 患者语音样本采集
   - 录音质量自动检测
   - 上传至 Supabase 存储

5. **无障碍**
   - 键盘快捷键全覆盖
   - 屏幕阅读器 ARIA 标注
   - 运动障碍适配（大按钮、高对比度）

### P2 — 增强功能

6. **微信小程序**
   - 核心识别功能的小程序版本

7. **数据可视化**
   - 使用统计、识别准确率趋势

## 非功能需求

- **性能**：语音识别响应时间 < 3秒
- **可用性**：ASR 服务不可用时自动降级，不阻断使用
- **安全**：用户录音数据加密存储，认证使用 Supabase Auth
- **兼容性**：Chrome 90+, Safari 15+, 微信内置浏览器
